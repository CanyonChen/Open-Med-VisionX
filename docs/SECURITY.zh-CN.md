# OpenMedVisionX 安全政策

[English](../SECURITY.md)

## 支持的版本

本项目目前尚未达到 1.0 版本。安全修复只应用于默认分支上的最新源代码。旧版本快照和经过本地修改的构建不在支持范围内。

## 私密报告

遇到以下情况时，请勿创建公开 Issue：

- 可导致代码执行、路径遍历、数据暴露、凭据泄露或拒绝服务的漏洞；
- 在源代码或提交历史中发现的 API 密钥、私钥、访问令牌或密码；
- 在仓库或任何产物中发现的 DICOM、NIfTI、像素数据、元数据，或其他可能属于患者的数据；
- 恶意插件，或权限超出预期的插件。

请使用该仓库的 GitHub 私密漏洞报告功能。如果该功能不可用，请通过仓库资料页上显示的私密渠道联系维护者。请勿附加敏感文件；只提供能够复现问题的最小合成示例，并说明维护者可以在哪里进行私密核查。

一份有效的报告应包含受影响的版本或提交、运行平台、影响范围、使用合成数据的复现步骤，以及任何建议的缓解措施。

## 数据暴露后的立即响应

如果密钥或凭据已经暴露：

1. 立即撤销或轮换相关密钥或凭据。
2. 停止使用受影响的凭据，并检查服务提供方的审计日志。
3. 从工作树中移除相关内容。
4. 重写 Git 历史，并清除已发布的产物或缓存，然后才能再次公开仓库。
5. 新增或加强自动检测规则。

如果可能的医学数据或患者数据已经暴露：

1. 停止分发并限制访问。
2. 除了识别受影响对象所必需的范围，不要复制、检查或再次分发更多数据。
3. 从项目控制范围内的工作树、Git 历史、发布版本和镜像中移除相关内容。
4. 通知负有责任的数据所有者，并遵循适用的机构和法律事件处理程序。
5. 将所有依赖这类数据的测试替换为运行时生成的合成样例。

在后续提交中删除文件，并不能将它从 Git 历史中移除。

## 安全边界

- Python 模型适配器能够执行任意代码。独立进程和 Conda 环境可以改善故障隔离，但并不是安全沙箱。
- 导入的模型文件、适配器和权重在用户审查其来源与许可证之前都应视为不可信。三个内置参考和一个教学样例遵循另一份经过维护者审查的允许列表，其中固定记录了来源、许可证、字节大小、哈希值和总量预算；这项审查并不意味着它们的输出具有临床有效性。
- 网络使用与图像附加默认关闭。图像还需要最终检查，并针对精确的 provider、endpoint、model、task、prompt 与 PNG 进行一次性授权。已经完成的传输无法召回。
- 只能向服务提供方发送用户检查过的精确新编码渲染 PNG。原始 DICOM/NIfTI、完整序列、源路径和 DICOM 元数据不会作为服务载荷。
- 即使移除了元数据，烧录在像素中的文字仍可能暴露身份信息。
- 本应用不是医疗器械，不适用于临床决策。

## 安全保存与分享文件

- 将患者数据、凭据、用户提供的模型、临时文件与导出内容保存在 OpenMedVisionX 源码目录之外。
- 在上传被拒绝的输入、崩溃产物、截图或实验记录前，先检查其中是否含有标识符、密钥、私有路径或受限内容。
- 三份经过审查的模型 / golden 包与公开 LoDoPaB-CT 教学样例，是源码发行包中仅有的已记录二进制数据例外；它们的存在并不表示可以加入其他模型或数据集。
- 删除文件名、过滤元数据、哈希和自动扫描可以降低风险，但不能证明文件或图像已经匿名化或适合分享。
