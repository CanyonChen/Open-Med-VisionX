# OpenMedVisionX 学习课程

[English](TEACHING_CURRICULUM.md) · [文档索引](INDEX.zh-CN.md) · [用户指南](USER_GUIDE.zh-CN.md) · [数据格式](DATA_FORMATS.zh-CN.md) · [内置模型](MODEL_BUNDLES.zh-CN.md) · [自定义模型](CUSTOM_MODELS.zh-CN.md) · [AI 安全](LLM_SECURITY.zh-CN.md) · [术语表](GLOSSARY.zh-CN.md)

本课程把桌面应用组织成一条循序渐进的学习路线：先理解像素与物理几何，再掌握 CT
重建和评价，最后把同样严谨的思路用于本地模型与可选 AI 助手。整个过程不需要编程。

> [!WARNING]
> 所有活动仅用于学习或研究。不得把输出解释为诊断，也不得用于治疗、分诊或任何
> 患者相关决策。只能使用你有权处理的数据，并优先选择不含敏感信息的示例。

## 路线与预计时间

开始前请完成[基础应用配置](QUICKSTART.zh-CN.md)。完整路线包括结课项目，
预计需要 **7–12 小时**。建议分成三次完成：基础（模块 1–3）、科学推理（模块 4–5）、
负责任的模型使用（模块 6–7 与结课项目）。

| 阶段 | 核心问题 | 主要工作区 | 预计时间 | 学习产出 |
| --- | --- | --- | ---: | --- |
| 1. 像素与显示 | 屏幕显示的是源数值，还是显示映射？ | 影像、教学 | 30–45 分钟 | 显示与源数据对照笔记 |
| 2. 医学几何 | 切片在什么条件下构成物理体数据？ | 影像、教学 | 45–60 分钟 | 几何记录表 |
| 3. CT 投影 | 角度、滤波器和迭代怎样改变重建？ | CT 实验 | 60–90 分钟 | 受控重建对比 |
| 4. 任务与输出 | 每类医学影像任务接收什么、返回什么？ | 教学、模型 | 45–60 分钟 | 任务契约图 |
| 5. 评价 | 拆分、指标、阈值与单位是否匹配问题？ | 评价、CT 实验 | 45–60 分钟 | 评价笔记与无像素记录 |
| 6. 本地模型推理 | 另一位用户能否复现输入和输出映射？ | 模型 | 75–120 分钟 | 内置模型实验笔记 |
| 7. AI 辅助解释 | 哪些内容离开本机，响应能够支持什么？ | AI 助手 | 30–45 分钟 | 网络边界检查记录 |
| 结课项目 | 整项结论能否被复现并明确限定？ | 一个或多个工作区 | 2–4 小时 | 简洁实验报告 |

模块 1、3、4、5 可以使用内置或普通非敏感示例。模块 6 的 DIVal 与 DeepInverse
活动也不需要用户数据集。DICOM、NIfTI、BraTS、自定义模型和图像发送分支都是可选项；
没有合适且获准使用的材料时必须跳过。

配置细节请查阅对应指南，不在课程中重复：

- [用户指南](USER_GUIDE.zh-CN.md)：工作区控件与端到端操作；
- [数据格式](DATA_FORMATS.zh-CN.md)：DICOM、NIfTI、普通图像、mask 与标注契约；
- [内置模型指南](MODEL_BUNDLES.zh-CN.md)：PyTorch 配置与三条经过审查的流程；
- [自定义模型指南](CUSTOM_MODELS.zh-CN.md)：本地清单与能力边界；
- [AI 安全指南](LLM_SECURITY.zh-CN.md)：启用任何网络请求前必须阅读；
- [术语表](GLOSSARY.zh-CN.md)：遇到不熟悉的术语或单位时查阅。

## 使用同一份学习记录

每个模块都使用同一份简短记录。这样可以比较参数变化、假设与限制，而不是最后只留下
截图。

| 字段 | 记录内容 |
| --- | --- |
| 问题与预测 | 一个可以回答的问题，以及运行前的预期 |
| 来源与权限 | 输入来自哪里、许可或获准用途，以及它是合成、基准还是本地数据 |
| 分析单位 | 按需写明 pixel、image、series、volume、lesion、slide、study 或 person |
| 输入契约 | modality、shape、dtype、范围／单位、几何、通道顺序与预处理 |
| 受控变化 | 唯一被比较的参数、阈值、算法或模型 |
| 输出与指标 | 每项输出的含义、所用参考，以及指标的数据范围或单位 |
| 失败或其他解释 | 一个失败位置，或观察现象的另一种可能原因 |
| 证据边界 | 本活动**不能**证明什么 |

不要把凭据、标识符、源路径或敏感像素复制到学习记录中。如果截图确有帮助，保存或
分享前也要像检查原始数据一样仔细检查截图。

## 1. 像素、dtype 与显示

### 学习目标

- 区分存储数组与渲染后的外观。
- 说明 shape、dtype、bit depth、数值范围、颜色／通道顺序、alpha 与压缩是相互独立
  的属性。
- 识别哪些显示操作只改变可见效果，不改变源数值。

### 准备

- 使用一份你有权检查的本地 PNG、JPEG 或 TIFF。
- 如果条件允许，从同一个非敏感源准备无损 PNG/TIFF 与 JPEG。
- 阅读[数据格式中的普通图像章节](DATA_FORMATS.zh-CN.md)。

### 操作

1. 在“影像”中打开图像，记录 shape、dtype、源范围与颜色解释。
2. 打开直方图，找出最小值、最大值、背景与看起来可能截断的区域。
3. 每次只改变一项：亮度、对比度、gamma、窗或显示范围。
4. 恢复原始显示，确认记录的源 dtype 与范围没有变化。
5. 在正常比例及锐利边缘、低对比区域比较无损版本与 JPEG。
6. 如果图像含颜色或 alpha，指出通道顺序，并说明 alpha 属于测量数据还是显示状态。

### 应观察到什么

- 同一个数组经过不同显示映射后，外观可能差异很大。
- 两个数组在 8-bit 屏幕上可能看起来相似，但保留的数值精度不同。
- 即使肉眼难以看到伪影，JPEG 也可能造成不可逆变化。
- 直方图描述存储值或选定值，但不能单独证明 modality、单位或临床含义。

### 自检

1. 不依赖屏幕颜色，你能否说出源 dtype 与范围？
2. 哪些控件只改变显示，哪类操作会生成派生像素数组？
3. 为什么灰度外观和文件名都不能证明数值是 HU？
4. 类别 mask 应使用哪种插值？为什么？

### 常见误区

- 把普通灰度值称为 HU。
- 把屏幕上的 8-bit 渲染当作源数组。
- 把 DPI 当作医学 pixel spacing。
- 对类别标签使用双线性插值。
- 认为视觉上干净的 JPEG 就是无损图像。

## 2. 页面、体数据与坐标

### 学习目标

- 区分二维普通图像、页面序列与物理体数据。
- 说明为什么只有具备 spacing、origin、orientation/direction 和 affine 或等价几何，
  毫米与正交视图才有意义。
- 区分源几何、显示状态与经明确确认的派生重采样。

### 准备

- 使用多页 TIFF 完成页面序列练习。
- DICOM 或 NIfTI 是可选项，必须来自获准使用的本地来源。NIfTI 支持需要安装
  [数据格式](DATA_FORMATS.zh-CN.md)所述的可选组件。
- 如需练习标签，请使用匹配的无损 label map，或引用当前已打开影像序列的 DICOM
  SEG/RTSTRUCT。

### 操作

1. 打开多页 TIFF，记录页面导航方式，以及是否存在可信 spacing、origin 或 orientation。
2. 如果有获准数据，打开经过校验的 DICOM 或 NIfTI 体数据，记录尺寸、spacing、方向、
   modality 与 intensity meaning。
3. 依次查看 axial、coronal、sagittal，记录一个 voxel index、对应物理坐标，以及连接
   各视图的变换。
4. 比较页面导航与正交体数据导航。
5. 如果 NIfTI 含第四轴，显式选择一个 volume 并记录 index；不要把它描述为完整 4D
   播放或分析。
6. 在“图层”中识别基础图像／体数据、segmentation layer 与 RTSTRUCT contour layer。
   区分源几何与 opacity、颜色、可见性、活动图层等状态。
7. 如果有合适数据，导入匹配 label map 或已引用的 DICOM 标注，确认匹配几何可见且
   可以检查。
8. 只能用非敏感测试数据检查故意错配的网格。确认源数据保持不变，并且 overlay 在你
   明确批准派生显示重采样前保持隐藏或 pending。

### 应观察到什么

- 仅凭 TIFF 页面顺序不能建立患者空间几何。
- 同一个物理点在不同正交平面中可以对应不同 voxel index，但仍指向同一位置。
- DICOM series 选择与切片顺序依赖标识和几何，不只依赖文件名。
- NIfTI 会统一到便于查看的方向，但容器本身不能证明 modality 或强度单位。
- 离散标签重采样使用 nearest-neighbour，并且只是派生显示；原始标签不会被静默改写。

### 自检

1. 哪些证据能够证明对象是物理体数据，而不只是页面堆栈？
2. spacing 与 orientation 来自哪里，是源数据提供还是用户提供？
3. 为什么数组轴不能自动等同于 axial、coronal 或 sagittal？
4. 负责任地叠加 DICOM SEG 或 RTSTRUCT 前，哪些标识与几何必须一致？
5. 明确重采样后，什么发生了变化，什么仍然不可变？

### 常见误区

- 只按文件名排序 DICOM。
- 在没有 affine 时根据数组顺序推断物理方向。
- 把多页 TIFF 当作医学体数据。
- 静默缩放 mask 来匹配参考。
- 认为每份 NIfTI 都是 CT、MR 或以 HU 计量。

## 3. 从物体到 sinogram，再返回图像

### 学习目标

- 说明 Radon 变换、角度采样、反投影、滤波与迭代校正。
- 在固定参考与显示量程的条件下比较 FBP 与 SART。
- 区分教学用图像域模拟与扫描仪原始数据。

### 准备

- 不需要输入文件。打开“CT 实验”，保留“合成仿体（推荐）”。
- 建立一个小表，记录角度范围、投影数量、滤波器、迭代次数、耗时与指标。
- 比较重建时始终使用同一参考与显示范围。

### 操作

1. 记录为什么合成仿体的非负值、说明性的 `mm⁻¹` 单位与零背景，比任意已打开图像
   具有更清晰的教学契约。
2. 固定角度范围和投影数量，选择“运行首个仿体实验”。
3. 比较无滤波反投影与滤波反投影。
4. 在不改变角度和参考的条件下比较至少两种 FBP 滤波器。
5. 使用多个迭代次数运行 SART。每次只改变迭代次数，记录质量、噪声、耗时与取消行为。
6. 检查一个 Radon 中间视图与差异／误差视图。
7. 只有记录了每项指标的数据范围和单位后，才比较 MSE、PSNR、SSIM 或其他指标。
8. 可选：检查高级活动图像模拟。记录阻止不兼容图像的 domain check，并解释即使像素
   被接受，它们仍是图像派生模拟，不是扫描仪原始采集。
9. 可选：运行[内置模型指南中的 LoDoPaB 示例](MODEL_BUNDLES.zh-CN.md)，
   比较 observation、固定 Hann FBP 与 reference，但不要把归一化衰减值称为临床 HU。

### 应观察到什么

- 稀疏或有限角度采样产生结构化伪影，而不是普通随机噪声。
- 无滤波反投影会模糊投影贡献；滤波能改善分辨率，也会改变噪声和 ringing。
- 增加 SART 迭代开始时可能改善匹配，随后可能放大噪声或增加耗时而没有实质收益。
- 各自独立自动拉伸会隐藏差异，因此必须共享参考量程。
- 视觉上更顺眼的重建不一定具有最低数值误差。

### 自检

1. 每次比较中只改变了哪一个参数？
2. 为什么 360° 平行束采样会重复 180° 已包含的信息？
3. 每项指标对应什么 domain 和 data range？
4. 能否从显示重建追溯到滤波器、投影几何与源物体？
5. 为什么合成示例与 LoDoPaB 示例都不是扫描仪原始数据，也不能证明临床性能？

### 常见误区

- 把重建后的 HU 切片当作原始衰减测量。
- 在各自独立自动拉伸后比较输出。
- 同时改变角度数量、滤波器、算法与显示范围。
- 认为迭代越多，重建一定越好。
- 把一个公开基准案例当作泛化证据。

## 4. 医学计算机视觉任务地图

### 学习目标

- 识别常见医学影像任务的输入、输出、baseline 与评价单位。
- 区分模型家族名称与可执行输入契约。
- 追问哪些采集、人群、标签与验证能够支持预期用途。

### 准备

遇到不熟悉的术语时使用[术语表](GLOSSARY.zh-CN.md)。下表的模型家族用于学习，
不代表当前桌面一定能够运行。

| 任务 | 输入 → 输出 | 简单 baseline | 代表性家族 | 评价重点 |
| --- | --- | --- | --- | --- |
| 分类 | 图像／体数据 → 标签分数 | 逻辑回归或小型 CNN | ResNet、DenseNet、EfficientNet、ViT | 按 person 拆分、校准、阈值选择 |
| 分割 | pixel／voxel → label map | 阈值或区域生长 | U-Net、nnU-Net、UNETR、提示式 SAM 变体 | Dice 加边界与小区域误差 |
| 检测 | 图像 → box／point／class | 连通域 | Faster R-CNN、RetinaNet、YOLO、DETR | IoU、匹配规则、按 lesion 或 image 评价 |
| 配准 | moving + fixed image → transform／field | 刚体或仿射优化 | VoxelMorph、TransMorph | landmark、folding、逆一致性、插值 |
| 重建 | 测量 → 图像 | FBP、SART、zero-filled IFFT | FBPConvNet、learned primal-dual、MoDL、VarNet | 测量一致性与采集假设 |
| 修复 | 退化图像 → 修正图像 | 经典滤波 | U-Net、Restormer、扩散方法 | 参考一致性与伪造纹理 |
| WSI 分析 | tile／feature → slide 结果 | 特征统计 | attention MIL、CLAM、TransMIL | 按 slide/person 拆分与 tile 采样 |
| 多模态／VQA／报告 | 图像 + 文本 → 文本／结构化输出 | 检索或模板 | CLIP 类编码器、VQA 与报告模型 | 证据定位、无依据结论、隐私、人工复核 |

### 操作

1. 从表中选择三类任务，其中至少一类是“测量到图像”或会改变几何的任务。
2. 针对每项任务，写出一个合理模型所需的精确 modality、dimensionality、channel
   order、spacing/orientation、dtype、range 与 normalization。
3. 定义每个输出 tensor 或对象，并说明怎样映射回源几何。
4. 先写简单 baseline，再写学习型模型。
5. 选择评价单位与至少两项互补指标。
6. 说明什么人群和标签产生训练目标，以及预期用途需要哪类外部验证。
7. 分别记录代码、权重与数据许可。

### 应观察到什么

- 相似架构名称可以代表不同任务与互不兼容的预处理。
- 输出几何是任务契约的一部分，不是事后显示细节。
- 高层协议可以描述比当前桌面安全收集能力更多的输入。
- baseline 能揭示学习方法是否真的优于更简单的方法。
- AUROC、Dice 与视觉质量回答不同问题，不能相互替代。

### 自检

针对一个选定任务，在不使用“模型接受什么就输入什么”的前提下回答：

1. 输入单位与输出单位分别是什么？
2. 哪些几何和强度假设必须成立？
3. 每个输出值表示什么？
4. 哪个 baseline 与哪项指标匹配科学问题？
5. 哪类人群、标签与拆分定义了现有证据？
6. 哪项预期用途仍未得到支持？

### 常见误区

- 把架构名称当作完整任务定义。
- 认为二维图像可以替代三维体数据、k-space 或 sinogram。
- 只用 Dice 评价边界或小病灶。
- 没有参考值就把合理修复纹理称为恢复出的解剖结构。
- 认为协议层支持就等于当前 GUI 能收集所有必需输入。

## 5. 避免数据泄漏的评价

### 学习目标

- 选择与科学问题匹配的拆分、阈值、指标与不确定性汇总。
- 区分 discrimination、calibration、overlap、boundary error 与 image fidelity。
- 区分单病例演示与数据集级泛化。

### 准备

- 使用“评价”中的内置示例与一项已保存 CT 实验结果。
- 数据集 manifest 是可选项。如需使用，必须是假名化记录，学习笔记中不得暴露标识符
  或敏感路径。
- 查看分数前先决定评价单位：pixel、lesion、image、series、volume、slide、study
  或 person。

### 操作

1. 加载概率前，先写出预测目标、评价单位与预期拆分。
2. 如果使用 manifest，校验同一 group 绝不跨越 train/validation/test，并确保预处理
   只在训练数据上拟合。
3. 在“评价”中输入二值标签与概率，使用两个显式阈值运行，记录 confusion matrix
   计数与派生指标中哪些发生变化。
4. 将依赖阈值的指标与 AUROC、calibration、可用置信区间比较，说明哪些量不依赖当前
   operating threshold。
5. 对 CT 实验结果，先确认共享几何与声明 data range，再解释 MSE、PSNR 或 SSIM。
6. 对分割任务，适用时将 Dice 等区域重叠指标与 HD95 等边界距离及体积或病灶级指标
   配合使用。
7. 检查至少一个失败案例或 subgroup。只有数据与权限允许时，才记录站点、扫描仪、
   协议、人群与图像质量因素。
8. 导出可用的不含像素 `ExperimentRecord`，确认它记录设置与指标而不是源像素。

### 应观察到什么

- 改变阈值会改变预测类别、sensitivity/specificity 与相关计数，但不会改变原始概率
  的排序。
- AUROC 不能证明校准良好，也不能确定临床可接受的 operating point。
- 按切片随机拆分可能使同一 person 或 study 的信息进入多个分区。
- 良好平均值可以掩盖边界误差、小区域失败或有害 subgroup。
- 置信区间描述抽样不确定性，不能覆盖所有偏差或域偏移来源。

### 自检

1. 是否在切片、patch、增强或特征拟合前完成了拆分？
2. 阈值是否在不使用保留测试集的前提下选择？
3. 每项指标是否都有声明的单位、范围、参考与评价单位？
4. 哪项互补指标能暴露 headline metric 隐藏的失败？
5. 哪类人群或采集偏移没有被本次评价覆盖？

### 常见误区

- 在测试集上选择阈值，再把同一结果当作无偏估计。
- 把 AUROC 等同于 calibration 或临床效用。
- 把按切片拆分报告成按 person 泛化。
- 比较使用不同 data range 计算的 PSNR。
- 把一个基准案例或一份导出记录当作外部验证。

## 6. 可复现的本地模型推理

### 学习目标

- 在不隐藏输入含义、预处理、设备回退或输出映射的前提下运行经过审查的本地模型。
- 解释为什么每份内置模型都需要专用输入路径。
- 识别自定义模型能力门禁在何时正确拒绝契约。

### 准备

- 根据[内置模型指南](MODEL_BUNDLES.zh-CN.md)安装并验证 PyTorch；不要在这里重复或
  猜测平台专用命令。
- DIVal 与 DeepInverse 不需要用户数据集。MONAI 需要获准使用的本地 BraTS 风格案例；
  没有合适案例时可以只学习契约，不运行模型。
- 用户自备模型是可选项。如需使用，请按[自定义模型](CUSTOM_MODELS.zh-CN.md)准备，
  并检查来源与许可。

| 经审查的参考 | 输入边界 | 输出边界 |
| --- | --- | --- |
| [DIVal LoDoPaB FBP-U-Net](../src/workbench/resources/model_bundles/dival-lodopab-fbpunet/MODEL_CARD.md) | 随包固定 LoDoPaB Hann 滤波 FBP，`1×1×362×362`，归一化衰减；不是任意 CT、临床 HU 或扫描仪 sinogram | 一份基准域后处理重建；单病例指标不是临床性能 |
| [DeepInverse MRI MoDL 教学模型](../src/workbench/resources/model_bundles/deepinv-mri-modl/MODEL_CARD.md) | 本地生成的确定性合成 MRI：模拟单线圈复数 k-space 与二值 mask；不是扫描仪原始数据 | 双通道复数重建，幅值只用于派生显示；演示当前不支持 JSON 导出 |
| [MONAI BraTS 三维分割](../src/workbench/resources/model_bundles/monai-brats-segmentation/MODEL_CARD.md) | 用户管理且完成配准的 T1/T1ce/T2/FLAIR/SEG NIfTI，满足规定几何与标签；项目不附带 BraTS 案例，也不自动配准 | WT/TC/ET 与参考指标；BraTS 2018 → 2021 是明确域偏移，不是通用病灶或临床验证 |

### 操作

1. 打开“**模型 → 内置模型**”，选择“**验证目录与设备**”，记录可用设备，但不要把
   验证解释为准确性证明。
2. 运行“**DIVal LoDoPaB FBP-U-Net**”，比较 observation、经过审查的 Hann FBP、
   模型输出、真值、固定量程误差与 MAE/RMSE/PSNR；需要时导出不含像素的实验 JSON。
3. 运行“**DeepInverse MRI MoDL 教学模型**”，沿数字模体、单线圈 FFT、固定欠采样、
   零填充 IFFT、MoDL 幅值图与绝对误差追踪流程，并始终把整条路径标记为合成 MRI。
4. 阅读 MONAI 模型卡。如有获准数据，使用“**配置本地 BraTS 2021…**”校验一份
   T1/T1ce/T2/FLAIR/SEG 案例，再运行已启用的完整体流程，检查 WT/TC/ET、Dice、
   HD95 与体积误差。没有合适案例时，只记录契约，不选择替代数据。
5. 可选：打开“**模型 → 外部清单**”，从文档示例开始，用实际本地模型事实替换每个
   placeholder，而不是只修改名称。
6. 加载自定义模型前，检查 task、modality、input count、dimensionality、layout、
   channel order、range、normalization、空间变换、输出、标签、runtime、来源与许可。
7. 阅读能力判断。只有当前桌面能提供全部输入时才运行；如果契约需要 prompt、复合输入
   或完整 3D/4D 数据，把拒绝结果记录为有用结论。
8. 每次运行都记录请求与实际设备、回退、模型版本、预处理、耗时、警告、输出含义与
   参考指标。

### 应观察到什么

- 三条内置流程都理解任务，不会把任意活动图像变成看似合理但无效的输入。
- DIVal 使用经过审查的公开基准案例；DeepInverse 在本地生成确定性输入；MONAI 在有效
  本地案例通过本次会话校验前保持禁用状态。
- 显示设备可能在明确回退后变为 CPU；可复现记录必须保存实际设备，而不只记录请求设备。
- 外部清单路径当前只接受一个受支持的活动二维图像或平面，不能借用 DeepInverse 的
  复合输入或 MONAI 的四体流程。
- 看似有效的输出仍可能抑制、偏移或虚构结构。

### 自检

1. 对 DIVal，为什么归一化衰减与固定 FBP 不是临床 HU 或任意扫描仪投影？
2. 对 DeepInverse，k-space 来自哪里？为什么必须持续标记为模拟？
3. 对 MONAI，必须记录哪些模态、几何、标签与域偏移？
4. 对自定义模型，当前桌面能否在不虚构或静默变换输入的前提下收集所有声明输入？
5. 另一位用户能否根据记录复现预处理、实际设备、输出映射、阈值与参考？

### 常见误区

- 认为 `.pt`、`.pth` 或 `.ckpt` 扩展名可以证明文件安全且契约完整。
- 把当前二维平面输入给在完整三维体数据上训练的模型。
- 只为匹配 tensor 而改变 resize、通道顺序、模态顺序或 normalization。
- 把 CPU/GPU 兼容性检查或 golden 结果当作医学准确性。
- 把能力拒绝当作模型故障，而不是输入安全边界。

## 7. 负责任的 AI 辅助解释

### 学习目标

- 区分有帮助的解释与科学或临床证据。
- 在发出请求前，使网络边界、精确载荷、目的地与同意状态可见。
- 识别固化文字与请求已发出等残余隐私风险。

### 准备

- 完整阅读[AI 助手、隐私与云端图像](LLM_SECURITY.zh-CN.md)。
- 准备不含报告文字、标识符、凭据或敏感上下文的纯概念 prompt。
- 图像发送是可选项。只能使用明确不敏感、且你有权发送给所选服务商的图像。
- 在检查 endpoint、model ID、credential reference 与预期任务前，保持网络访问关闭。

### 操作

1. 使用精确 endpoint、精确 model ID，以及环境变量或 keyring credential reference
   配置服务。不得把原始 key 粘贴到 reference 字段。
2. 只为计划中的请求启用网络。发送纯概念 prompt，并记录 provider、endpoint host、
   model ID、时间与任务。
3. 将响应与可见实验及可信来源比较，标出无依据、含糊或无法验证的说法。
4. 如果考虑图像请求，启用 vision 并附加活动渲染平面，继续前检查规范 PNG preview。
5. 在最终检查中核对 provider、精确目的地、model、task、prompt hash、image hash、
   dimensions、bytes、display transform、最小化操作、固化文字检查与残余风险。除非每项
   都正确且适合发送，否则保留默认答案“**否**”。
6. 修改一项计划字段，确认必须重新检查。记住，同意是一次性的，只用于一次精确发送。
7. 将“**结构化产物**”作为集成预览检查。确认或拒绝产物只记录本地 review；不会发送
   数据、创建图像图层或证明产物有效。

### 应观察到什么

- 文本请求发送完整 prompt，而不只是选定关键词。
- 图像请求发送完整活动二维平面的新编码渲染，不发送原始 DICOM/NIfTI 或完整 series；
  这种最小化仍不能移除固化在像素中的文字。
- provider、endpoint、model、prompt、image 或 task 变化都会使先前检查失效。
- 取消请求不能召回已经到达服务商的 bytes。
- 流畅文字仍可能错误、缺少依据或超出模型能力。

### 自检

1. 哪些精确文本和图像 bytes 会离开本机？
2. 哪个 host 会接收它们？对应哪项服务商政策与 model ID？
3. 哪些标识、metadata、overlay 和固化像素仍保留或已移除？
4. 响应中的哪项说法由实验支持，哪项还需要其他来源？
5. 如果意外发送敏感数据或凭据，应怎样处理？

### 常见误区

- 把报告、标识符或原始 API key 粘贴到 prompt 或设置字段。
- 把流畅文字当作经过验证的解释。
- 认为同意可以自动扩展到另一服务商、模型、prompt、任务或图像。
- 认为移除 metadata 也会移除固化文字。
- 认为取消或撤销可以召回已发送请求。
- 把结构化产物 review 当作自动导入或临床批准。

## 结课项目

### 学习目标

- 组合一个清晰问题、有效输入契约、受控比较、适当指标与有边界的结论。
- 生成另一位用户可以复现和质疑的记录。
- 不隐藏失败或含糊结果，明确写出隐私、偏差、域偏移与临床限制。

### 准备

选择一条不含敏感信息的路线：

- 比较两组 CT 实验重建设置；
- 分析 DIVal 或 DeepInverse 内置演示；
- 使用两个阈值评价标签与概率；
- 只有具备获准且符合契约的数据时，才运行 MONAI 或自定义本地模型；
- 审计纯概念 AI 解释及其网络计划。

运行前写下问题与预测结果。准备相关用户指南、数据／模型卡、来源／许可记录，以及一份
空白学习记录。

### 操作

1. 定义问题、预期学习用途，以及本活动确实有能力检验的一项 claim。
2. 记录来源、权限、排除标准、评价单位、输入输出含义、几何与预处理。
3. 选择简单 baseline，以及一个受控算法、模型、参数或阈值比较。
4. 如果使用数据集，在切片、patch、增强或预处理拟合前定义 group-safe split。
5. 运行流程，记录参数、实际设备、耗时、警告、显示映射、参考，以及带单位／范围的指标。
6. 检查一个失败案例、异常区域或其他解释。
7. 说明仍然存在的隐私、偏差、采集、人群、域偏移与临床限制。
8. 写出严格限定在证据范围内的结论。如果结果失败或无法下结论，也应如实保留。
9. 组织一份简洁报告，包含可复现步骤，且只使用不敏感、确有必要的图示或无像素导出。

### 应观察到什么

- 狭窄且受控的问题，比宽泛的“模型是否有效”更容易产生清晰结论。
- 输入契约与显示选择对解释的影响，可能与算法选择同样大。
- 失败案例通常比最好看的输出更能揭示有效 domain。
- 可复现性需要参数、单位、几何、版本与警告，不能只保留截图。

### 自检

1. 另一位用户能否在不猜测隐藏设置的前提下重复流程？
2. baseline 是否公平？是否只改变了预期因素？
3. 每张图与每项指标是否都有参考、单位／范围和输出含义？
4. 拆分、阈值与预处理选择是否独立于保留结果？
5. 结论是否说明证据不能证明什么？
6. 报告是否排除了敏感像素、标识符、路径与凭据？

### 常见误区

- 把视觉上令人印象深刻的输出当作充分证据。
- 隐藏失败运行，或不断改变多项设置直到结果好看。
- 把单个案例称为数据集级或临床验证。
- 报告指标时不说明参考、单位、范围或评价单位。
- 写出超出实验设计支持范围的结论。

## 课程产出

完成路线后，应保留以下不含敏感信息的产出：

1. 七份完整学习记录，每份包含自检答案与一项证据边界；
2. 一份像素／显示对比与一份页面／体数据几何记录；
3. 一份受控 CT 重建对比；
4. 一份医学影像任务契约图；
5. 一份阈值／评价分析，并在可用时附带无像素记录；
6. 一份内置模型实验笔记；只有具备合适获准输入时，才附加 MONAI 或自定义模型工作；
7. 一份 AI 网络边界检查记录，也可以是不发送数据的计划；
8. 一份结课报告，包含问题、契约、受控结果、失败案例、可复现信息与证据限定结论。

当另一位用户能够复现工作、解释输入为何符合科学要求、指出发生了什么变化，并且不依赖
视觉上可信的输出来说明限制时，课程才算完成。下一项用户任务请返回[文档索引](INDEX.zh-CN.md)。
