# AI 助手、隐私与云端图像

[English](LLM_SECURITY.md) · [文档首页](INDEX.zh-CN.md) · [用户指南](USER_GUIDE.zh-CN.md#ai-助手)

“**AI 助手**”可以向您配置的服务请求医学计算机视觉概念、参数或实验结果解释。AI
服务是可选能力；本地影像查看、CT 实验、评价、教学模块和内置模型运行均不依赖它。

> [!WARNING]
> AI 服务响应只可作为学习辅助，不是诊断或经过验证的影像解释。文本与图像像素都可能
> 包含敏感信息。只发送您有权发送到所选目的地的精确数据。

## 先判断是否真的需要 AI 服务

只有在需要解释可见概念或结果，并且已检查服务条款、保留策略、处理地点和账户控制时，
才使用 AI 助手。不要仅仅因为界面中存在该工作区就启用它。

以下情况优先使用内置“**教学**”模块与本地文档：

- 学习课程已经覆盖该问题；
- 提示词会包含患者、机构或保密上下文；
- 无法确认服务在哪里处理或保留数据；
- 无权发送精确文本或像素；
- 答案可能影响患者相关决策。

OpenMedVisionX 启动时网络默认关闭。导入或打开本地数据不会联系 AI 服务。

## 什么内容可能离开您的电脑

系统有两条相互独立的传输路径：

| 操作 | 会发送什么 | 不会发送什么 |
| --- | --- | --- |
| 发送文本问题 | 完整提示词、请求所需对话消息，以及 provider/model 请求字段。 | 除非单独启用并授权图像附加，否则不发送本地图像。 |
| 附加活动渲染平面 | 完整活动二维平面经过显示映射后重新编码的 PNG，以及提示词和请求字段。 | 原始 DICOM/NIfTI 字节、DICOM 元数据、完整序列/体数据、源路径、视口缩放/平移、叠加层、测量与标注。 |

删除元数据并不能删除烧录在像素中的姓名、日期、检查号、机构文字或其他标识符。自由
文本同样可能泄露敏感信息。OpenMedVisionX 无法证明任一载荷已经匿名，必须由您检查。

桌面中的另一类网络操作是确认后打开外部 HTTP(S) 链接。本地工作流不会自动下载
数据集、模型或依赖。

## 配置服务且不在项目中保存密钥

打开“**AI 助手 → 教学对话**”，展开“**服务配置**”，完成以下字段：

1. **服务提供方（Provider）**——选择计划使用的协议/服务账户。
2. **模型 ID（Model ID）**——输入服务当前有效的精确模型 ID。OpenMedVisionX 不会替您
   选择或验证所谓“最新”模型。
3. **Endpoint**——检查精确目的地。远程地址必须使用 HTTPS。普通 HTTP 只允许精确
   本地回环地址，例如 `localhost`、`127.0.0.1` 或 `::1`。
4. **凭据引用（Credential reference）**——只填引用，绝不能填密钥本身。
5. **启用网络**——只有在目的地与提示词都准备好时才打开。

当前服务选项与默认凭据引用如下：

| 服务选项 | 默认协议地址 | 默认凭据引用 |
| --- | --- | --- |
| OpenAI | `https://api.openai.com/v1/responses` | `env:OPENAI_API_KEY` |
| Anthropic | `https://api.anthropic.com/v1/messages` | `env:ANTHROPIC_API_KEY` |
| Moonshot / Kimi | `https://api.moonshot.cn/v1/chat/completions` | `env:MOONSHOT_API_KEY` |
| 智谱 GLM | `https://open.bigmodel.cn/api/paas/v4/chat/completions` | `env:ZHIPU_API_KEY` |
| DeepSeek | `https://api.deepseek.com/chat/completions` | `env:DEEPSEEK_API_KEY` |
| OpenAI-compatible | 用户提供的兼容地址；初始 example.invalid 是不可用占位值 | `env:OPENMEDVISIONX_API_KEY` |

Endpoint 不得包含用户名、密码、query string 或 fragment。发送任何敏感请求前都要检查
显示的 host。

### 凭据引用

- `env:NAME` 从仓库外设置的环境变量读取密钥，例如 `env:OPENAI_API_KEY`。
- `keyring:service/username` 在安装 `python -m pip install -e ".[llm]"` 后从操作系统
  凭据存储读取。
- `none` 只允许用于明确无需认证、且位于精确回环地址的本地服务。

绝不能把真实密钥粘贴到“凭据引用”字段、YAML/TOML、源码、截图、日志、Issue 或聊天
记录中。该字段只填写引用字符串。

“**图像输入**”复选框只是您对所选模型支持图像的声明，不是自动能力探测。当前桌面会
把 DeepSeek 选项作为纯文本服务处理。

## 发送纯文本教学问题

先从不包含敏感上下文的问题开始：

1. 配置 provider、精确 Model ID、endpoint 与凭据引用。
2. 保持“**图像输入**”和“**附带活动渲染平面**”关闭。
3. 打开“**启用网络**”。
4. 输入聚焦的问题，例如“为什么 FBP 使用 ramp filter？”
5. 重新阅读完整提示词，删除标识符、私有路径、报告和不必要的案例细节。
6. 选择“**发送**”或按 `Ctrl/Cmd+Enter`。
7. 阅读响应元数据与仅供学习提示。结合可见实验和可信来源核验重要说法。

使用“**取消请求**”或 `Esc` 请求取消。取消可以终止仍在等待的读取，但无法召回已经
传出的文本。Markdown 响应中的链接只允许 HTTP(S)，且打开浏览器前还需再次确认。

**成功标准：**回答显示已配置的 provider/model，图像传输状态仍保持关闭。

## 只在最终检查后分享一张渲染图

图像使用比文本多一层门禁：必须已有影像、确认模型支持视觉，并分别主动选择
“**图像输入**”“**附带活动渲染平面**”和“**启用网络**”。

发送前，OpenMedVisionX 会创建传输计划并显示最终原生确认，其中包括：

- provider、精确 endpoint、目标 host 与 Model ID；
- 任务和提示词 SHA-256 指纹；
- 精确 PNG 预览、文件名、MIME type、尺寸、字节数与 SHA-256；
- 预览创建方式以及排除了哪些元数据；
- 烧录文字与服务保留策略等残余风险。

待发送 PNG 是**经过显示映射的完整活动二维平面**。它不是视口截图：缩放、平移、
叠加层、测量与标注均不包含。因此，确认对话框中的图像才是权威载荷；请检查每个角落
是否有烧录标识符。

授权默认选择“**否/取消**”，只适用于精确的
provider/endpoint/model/task/prompt/PNG 计划。任何绑定字段或像素变化都会使授权失效。
授权会在一次发送前立即封存和消费，无论该尝试成功与否；重试需要重新检查。有效 PNG
上限为编码后 8 MiB、16,777,216 像素。

只有在以下条件全部成立时才选择“**是**”：

- 您有权向该目的地发送精确像素与提示词；
- provider/model/host 正是预期值；
- 完整预览不包含禁止发送的可见信息；
- 任务确实需要图像输入；
- 您接受服务的保留与处理条款。

## 理解“结构化产物”预览

第二个标签页“**结构化产物 · API 预览**”说明七种类型化结果契约：`text`、
`class_scores`、`labels`、`mask_2d`、`mask_3d`、`reconstructed_image` 和
`reconstructed_volume`。

该标签页不是普通聊天结果转换器。当前版本中：

- 只有可信宿主集成可以提供已经类型化的请求与响应；
- 教学对话响应**不会自动转换为类型化产物**；
- **没有通往该预览的桌面产物导入器**，文件选择器也不能填充它；
- 普通纯桌面会话看到空状态属于预期行为；
- 只有类型化请求/响应精确匹配时，“**确认产物**”或“**拒绝产物**”才可用；
- 任一操作都只记录本地不可变审查，不发送数据、**不会创建图层**，也不会改变源数据。

确认只表示当前精确产物已在本地完成检查，不能证明真实性、临床有效性或后续使用许可。

## 取消、撤回或改变决定

- 发送文本前，关闭“**启用网络**”或不按“**发送**”即可。
- 图像发送前，在最终确认中选择“**否/取消**”。
- 已检查字段发生变化时，请检查新计划，不能沿用之前的决定。
- “**取消请求**”只会请求取消活动操作，不是数据召回机制。
- 关闭图像附加可以阻止未来请求加入图像，但无法删除服务已经接收的数据。

绝不能静默重试图像请求。重试是另一次披露尝试，需要新的精确检查。

## 处理凭据或数据事件

如果凭据可能已经泄露：

1. 立即在服务端撤销或轮换凭据；
2. 检查服务审计与计费日志；
3. 从本地文件、Git 历史、日志、截图与共享消息中移除凭据，且不要再次贴出原值；
4. 改用环境变量或 keyring 引用。

如果可能发送了患者信息或其他受限内容：

1. 停止后续请求并关闭网络；
2. 在可用时使用服务商的删除/事件处理流程；
3. 通过私密渠道通知负责机构或数据所有者；
4. 遵循适用法律与机构流程；
5. 不要在公开 Issue 中复现数据。

私下报告安全漏洞时请使用[安全策略](SECURITY.zh-CN.md)。

## 发送前检查表与下一步

每次发送文本前：

- [ ] provider、endpoint host 与精确模型均为预期值。
- [ ] 凭据字段是引用，而不是密钥。
- [ ] 完整提示词允许发送，且不含多余私密数据。
- [ ] 您会核验响应，而不是把它当作证据。

每次发送图像前：

- [ ] 已完成以上所有文本检查。
- [ ] 已向服务商确认模型的视觉能力。
- [ ] 已逐像素检查最终 PNG，而不只是查看视口。
- [ ] 一次性传输计划匹配预期任务与目的地。
- [ ] 您理解已完成的传输无法召回。
