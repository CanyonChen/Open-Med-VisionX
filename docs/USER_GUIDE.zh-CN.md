# OpenMedVisionX 用户指南

[English](USER_GUIDE.md) · [文档首页](INDEX.zh-CN.md) · [快速开始](QUICKSTART.zh-CN.md) · [数据与格式](DATA_FORMATS.zh-CN.md) · [故障排查](TROUBLESHOOTING.zh-CN.md)

本指南按照桌面应用从左到右的顺序展开，说明普通用户可以完成什么、每项操作需要什么、如何确认操作成功，以及应当遵守哪些科学与隐私边界。

> [!WARNING]
> OpenMedVisionX 仅用于学习和研究。所有输出都不是临床结论，不得用于诊断或患者相关决策。

## 开始之前

先完成[快速开始](QUICKSTART.zh-CN.md)，启动桌面应用，并保留启动终端，以便查看依赖或设备错误。应用默认显示英文；选择页眉中的 **中文** 可切换为简体中文，选择 **English** 可切回英文。切换语言不会清除当前影像、实验、表单值或结果。

页眉显示 **本地 · 研究**，含义如下：

- 本地影像和模型文件不会自动上传；
- CT、影像、教学、评估和内置模型流程可以在本地运行；
- 只有选中 **启用网络** 后，AI 助手才会发出请求；
- 发送影像还需要第二次主动授权，并逐次核对将要发送的确切 PNG；
- 打开官方网站或助手回复中的链接，都是另一次由用户主动发起的操作。

应用没有传统菜单栏，六个主要工作区以选项卡呈现：

| 快捷键 | 工作区 | 主要用途 |
| --- | --- | --- |
| `Alt+1` | **影像** | 打开、检查、测量、标注和导出影像。 |
| `Alt+2` | **CT 实验** | 使用仿体或活动平面学习投影与重建。 |
| `Alt+3` | **模型** | 运行经过审查的内置演示，或兼容的本地外部清单。 |
| `Alt+4` | **教学** | 按顺序学习六个结构化主题。 |
| `Alt+5` | **评估** | 校验数据集清单并评估二分类概率。 |
| `Alt+6` | **AI 助手** | 向教学服务提问，或检查结构化产物契约。 |

## 导航与活动视图

**影像**工作区可以显示一张普通图像、一组页面，或三维体数据的多个视图。当前可见视图中始终有一个**活动视图**。单击某个视图或让它获得焦点，即可将其设为活动视图。影像上方的上下文栏会显示活动视图名称及当前页码或切片位置。

活动视图很重要，因为后续功能会共用它作为输入：

- **CT 实验 → 活动影像（高级）**使用当前活动二维平面；
- **模型 → 外部清单**使用当前活动二维平面；
- 使用**面积 / ROI**绘制的区域可成为当前 CT 对比 ROI；
- **AI 助手 → 附加当前活动渲染平面**会发送经过显示映射后的完整活动平面。

对于几何信息有效的三维体数据，轴向、冠状和矢状视图采用 RAS+ 坐标并显示患者方向标记。在**平移 / 缩放**模式中双击某个位置，可让三个正交平面联动到同一点。悬停时会显示像素值；如果几何信息有效，还会显示以毫米为单位的 RAS+ 坐标。

活动渲染平面并不是视口截图。导出或发送给助手的 PNG 不包含视口缩放、平移、十字线、测量、导入叠加层或标注。

## 影像

### 用途

使用**影像**工作区打开本地普通图像、DICOM 来源或 NIfTI 体数据；检查解码后的数值与几何信息；浏览页面或正交平面；进行教学测量；配对只属于当前平面的叠加层；导入具有参考关系的临床图层；并保存渲染平面或不含像素的记录。

### 前置条件

应用启动时不要求预先打开影像。要读取 NIfTI，请先安装[快速开始](QUICKSTART.zh-CN.md)中说明的可选 NIfTI 依赖。只能使用你有权访问的数据，并保留源数据备份；查看器不会修改源文件。

### 步骤

1. **打开数据来源。**

   - 选择**打开影像 / DICOM ZIP**（`Ctrl+O`）可打开 `.dcm`、`.dicom`、`.zip`、`.nii`、`.nii.gz`、`.png`、`.jpg`、`.jpeg`、`.tif` 或 `.tiff`。
   - 选择**打开 DICOM 文件夹**（`Ctrl+Shift+O`）可打开包含 DICOM 检查的目录。应用只会在你选择的文件、文件夹或 ZIP 内查找。
   - 解码期间可选择**取消加载**安全请求取消。在新数据完整加载成功前，原先已加载的影像仍可继续使用；失败或取消不会替换它。

2. **处理存在歧义的医学影像来源。**

   - 如果 DICOM 来源包含多个序列，会出现**选择 DICOM 序列**对话框。比较状态、模态、描述、序列号、实例 / 切片 / 帧数、尺寸、像素间距、层厚、几何和警告。无法支持的行会被禁用。选中一个可用序列，再选择**打开所选序列**。
   - 该对话框不会显示患者字段、文件路径、原始 UID 或哈希值。
   - 如果 NIfTI 包含多个三维体，请在**时间点 / 通道（1 到 N）**中选择所需体数据；界面中的编号从 1 开始。

3. **浏览并检查数据。**

   - 单张二维图像只使用主视图。
   - 多页序列会显示**切片 / 页面**，在支持时还会显示**播放页面**。
   - 三维体数据会显示轴向 Z、冠状 Y 和矢状 X 滑块；可生成时还会显示轴向最大强度投影。
   - 使用鼠标滚轮缩放，在**平移 / 缩放**模式中拖动平移。选择**缩放至窗口**（`Ctrl+0`）可让所有可见视图适应窗口。
   - 选择**直方图**查看活动平面中解码后的数值分布。
   - 在**影像详情**中检查类型、来源类别、形状、数据类型、强度含义、可用能力、颜色 / 位深、间距、模态、原点、方向和经过隐私过滤的运行时元数据。

4. **只调整显示效果。**

   - 灰度数据提供**下限**和**上限**；具有有效 HU 含义的 CT 数据会将该控制标为 **HU 窗**。
   - 彩色数据提供 **RGB 亮度**、**RGB 对比度**和 **RGB 伽马**。
   - 选择**自动范围**可恢复适合查看的自动显示范围。

   这些控制只改变显示，不会改变解码数组，也不会改变其定量含义。

5. **在活动平面上测量或做标记。**

   - 选择**距离**，在两点之间拖动，然后在状态栏查看以 `px` 或 `mm` 表示的距离。
   - 选择**面积 / ROI**并拖出矩形，然后查看以 `px²` 或 `mm²` 表示的面积。数值影像还会显示均值和标准差。当前活动平面上至少为 3 × 3 像素的矩形，可供 CT 实验计算 ROI 指标。
   - 选择**标注**并单击，可放置点标记。
   - 按 `Esc` 可取消尚未完成的拖动。选择**清除标记**会移除交互标记，以及已明确配对的平面局部叠加层。
   - 普通图像在未知间距时只使用像素单位。只有从可信来源获得数值后，才选择**设置像素间距…**，输入正数的 **X 间距（mm）**和 **Y 间距（mm）**，再选择**应用**。OpenMedVisionX 绝不会根据 DPI 推断医学像素间距。

6. **选择正确的标注流程。**

   **平面局部教学叠加层**用于对某一个可见平面做快速对照：

   - **掩膜…**接受无损、单页、灰度 PNG 或 TIFF，其高和宽必须与活动平面完全一致。所有非零值都会显示为同一个二值掩膜，应用绝不会调整它的尺寸。
   - **标注…**接受不超过 4 MiB 的 UTF-8 JSON 文件。它必须使用左上角为原点的像素坐标，与活动平面尺寸一致，至少包含一个框或点，并且不能含未知顶层字段或外部引用。

   最小标注文件示例如下：

   ```json
   {
     "schema_version": 1,
     "coordinate_system": "pixel_xy_top_left",
     "image_size": [512, 512],
     "boxes": [
       {"x1": 20, "y1": 30, "x2": 120, "y2": 160, "label": "example"}
     ],
     "points": [
       {"x": 64, "y": 80, "label": "landmark"}
     ]
   }
   ```

   `image_size` 的顺序是 `[宽, 高]`。框坐标必须满足 `0 ≤ x1 < x2 ≤ 宽` 和 `0 ≤ y1 < y2 ≤ 高`，点坐标必须位于影像范围内。切换到另一页或切片时，平面局部叠加层会暂时消失；返回原平面后会再次出现。

   **参考检查图层**会针对已加载的 DICOM 或 NIfTI 序列进行校验，并在**图层**侧栏中管理：

   - 打开被引用的 DICOM 影像序列后，**DICOM SEG / RTSTRUCT…**可导入一个 `.dcm` 或 `.dicom` 标注文件。
   - 打开对应的 DICOM 或 NIfTI 参考影像后，**标签图…**可导入无损 PNG、无损 TIFF、`.nii` 或 `.nii.gz` 整数标签图。JPEG、彩色标签图、浮点标签存储和静默猜测参考关系都会被拒绝；四维标签图 NIfTI 必须明确选择一个体。
   - 如果导入数据的几何与显示网格不同，请检查来源形状、显示形状、差异项、插值方法和范围外取值。选择**创建重采样预览**会保留隐藏的原始图层，并增加一个可见的派生图层；选择**保留原始图层并隐藏**只保留未重采样的隐藏图层；选择**取消**则不做任何改变。离散标签使用最近邻插值，连续的分数数据使用建议的连续插值方法。
   - 在**图层**中选中一个图层，可查看类型、创建者、校验、来源格式、几何和修订信息。使用**可见**、**锁定**和**不透明度**（0–100%）调整显示。分割标签还各自具有可见复选框和文字标明的颜色值。基础影像不能通过这些显示控制编辑。

7. **导出派生结果。**

   - **导出 PNG…**会把经过当前灰度或彩色显示映射的完整活动平面写入一个新 PNG，不包含标记、十字线、掩膜、轮廓或图层叠加。
   - **保存记录…**会写入一个 JSON，其中包含平面选择、显示参数、配对标志和最新数值测量，不包含影像像素或源路径。

### 如何判断成功

- 状态栏显示 `已加载 <类型>: <形状>`，并且**影像详情**已有内容。
- 上下文栏显示预期的**活动视图**以及页面或切片位置。
- 平面局部导入会明确报告它已与当前平面配对。
- 参考图层导入后会出现在**图层**中，并显示校验和几何状态。
- 导出完成后状态栏会报告新文件名；如果需要核对确切像素，请使用其他查看器重新打开输出文件。

### 边界与注意事项

- 没有经过几何校验的页面序列不会被当作三维体数据。
- 超大普通图像可能只在内存中保留有界预览。警告会列出原始尺寸和显示尺寸；当前会话无法对其运行全分辨率推理。
- JPEG 可能包含有损伪影。显示调整无法恢复源信息，也不能创造 HU 或 SUV 含义。
- 切换页面或切片会清除交互测量，因为坐标参考已经改变。
- 平面局部叠加层与参考检查图层是两套不同流程。配对 JSON 框不会创建检查图层；导入临床图层也不等同于配对当前平面。
- 导出文件属于派生数据，应使用新保存位置。应用绝不会修改已打开的源文件。

## CT 实验

### 用途

使用 **CT 实验**学习 Radon 变换、反投影、滤波、直接傅里叶重建和 SART。推荐流程使用确定性、非负的合成衰减仿体，不需要任何医学文件。

### 前置条件

使用**合成仿体（推荐）**不需要额外数据。使用**活动影像（高级）**前，先在**影像**工作区打开影像并激活所需二维平面。活动平面必须为灰度、方形、有限值、非负且不是常数。如果选中**圆形支撑域**，重建圆之外的值还必须为零。

### 步骤

1. 选择**输入**：

   - **合成仿体（推荐）**会启用**仿体尺寸**，范围为 64–384 px、步长 32 px，默认 192 px。
   - **活动影像（高级）**会原样使用当前活动平面，作为数学意义上的图像域输入。

2. 设置模拟采集参数：

   - **范围 °**可选 `180` 或 `360`，默认 `180`。
   - **角度 / 投影数**范围为 8–1440，默认 180。
   - **圆形支撑域**默认选中。

3. 选择**算法**并设置当前可见选项：

   | 算法 | 选项 |
   | --- | --- |
   | `FBP` | **FBP 滤波器**：`ramp`、`shepp-logan`、`cosine`、`hamming` 或 `hann`。 |
   | `BP` | 没有额外参数。 |
   | `DFR` | **DFR 插值**：`nearest`、`linear`（默认）或 `cubic`。 |
   | `SART` | **SART 迭代次数**：1–100（默认 5）；**松弛因子**：0.01–1.00（默认 0.15）。 |

4. 运行流程：

   - 在仿体模式中选择**运行第一个仿真实验**。应用会生成仿体、模拟正弦图，并使用当前算法自动重建。
   - 在活动影像模式中选择**运行图像域模拟**；正弦图完成后，再选择 **2. 重建**。
   - 后台运行时可选择**取消**，安全请求终止操作。

5. 查看四个主要视图：**输入**、**正弦图**、**重建**和**绝对误差热图**。在**中间过程**中可按算法查看 Radon 投影快照、探测器频谱或滤波投影、BP/FBP 进度、DFR 频谱、SART 迭代、带符号归一化差异和绝对误差。

6. 阅读指标。输入和重建使用同一个评估范围。摘要包含 MSE、PSNR 和 SSIM；存在有意义的原始单位时，还包含 MAE、RMSE 和偏差。若**影像**工作区中有有效的当前**面积 / ROI**，还会增加 ROI 范围内的 MSE、PSNR 和 SSIM。

7. 选择**导出 PNG…**保存重建图，或选择**保存记录…**将来源模式、几何设置、算法选项、评估范围和数值指标保存为不含像素的 JSON。

### 如何判断成功

- 仿体模式会从**正在后台运行**进入**实验完成**，过程中不会要求选择影像。
- 正弦图先于重建出现；完整实验结束后四个视图都有内容。
- 指标行以**评估范围**开头，并显示 MSE、PSNR 和 SSIM。
- 只有重建结果存在后，**导出 PNG…**和**保存记录…**才会启用。

### 边界与注意事项

- 仿体采用示意性的 `mm⁻¹` 数值和固定 0–0.03 `mm⁻¹` 评估范围，不是扫描仪原始数据。
- 活动影像模式是图像域教学模拟。它不会把 HU 转为线性衰减系数、恢复扫描仪投影、静默转换 RGB、裁剪非方形影像或丢弃负值。
- 360° 平行束扫描包含冗余信息，应用会折叠并平均冗余的另一半。
- 修改仿体尺寸、角度范围、投影数或圆形支撑域，会同时使正弦图和重建失效；只修改算法选项会保留正弦图，但使重建失效。
- 显示效果更悦目并不证明重建更好。只能在相同评估范围内结合报告的指标比较结果。

## 模型

### 用途

**模型**工作区提供两条彼此独立的路径：

- **内置模型**展示三个经过审查的固定教学参考，包含可见的输入输出契约、模型卡、完整性信息、设备选择和专用演示。
- **外部清单**检查用户选择的 YAML 清单，加载它指向的现有本地模型，并对一个兼容的活动二维影像运行推理。

### 前置条件

安装目标流程需要的模型运行组件，详见[内置模型指南](MODEL_BUNDLES.zh-CN.md)。除非数据或权重自身的许可条款另有规定，请把用户管理的权重和数据集放在仓库之外。

外部模型需要经过核对的 `.yaml` 或 `.yml` 清单，以及清单引用的全部本地权重文件。MONAI 流程需要你通过官方流程取得一例有权使用的本地 BraTS 2021 数据；OpenMedVisionX 不会代为下载。

### 步骤

1. **选择内置模型执行设备。**在**内置模型**中，将**执行设备**设为**自动 · 优先 GPU**、**CPU**或 **CUDA GPU**。选择**校验目录与设备**可校验三个已登记的 SHA-256，并查看本地 Torch 设备。即使没有执行这项可选的会话级检查，每次模型加载仍会单独通过完整性门控。

2. **阅读三张模型卡，并运行相应的专用流程。**

   | 内置参考 | 运行方式与结果 |
   | --- | --- |
   | [DIVal LoDoPaB FBP-U-Net](../src/workbench/resources/model_bundles/dival-lodopab-fbpunet/MODEL_CARD.md) | 选择**运行 LoDoPaB 案例**。比较公开 LoDoPaB 观测、经过审查的 Hann FBP、FBP-U-Net 输出、真值和固定参考绝对误差。阅读 MAE、RMSE、PSNR、请求设备 / 实际设备、耗时、回退和警告。**导出实验 JSON…**会写入不含像素的可复现记录。 |
   | [DeepInverse MRI MoDL](../src/workbench/resources/model_bundles/deepinv-mri-modl/MODEL_CARD.md) | 选择**运行合成 MRI 演示**。该专用流程创建确定性数字仿体，模拟单线圈 k 空间和采样掩膜，并比较仿体、零填充 IFFT、MoDL 幅值与绝对误差。阅读 MAE、RMSE、PSNR、耗时、回退和警告。 |
   | [MONAI BraTS 三维分割](../src/workbench/resources/model_bundles/monai-brats-segmentation/MODEL_CARD.md) | 在一个本地案例通过校验前，MONAI 推理保持**禁用**。选择**设置本地 BraTS 2021…**并完成引导式校验，再选择**运行 BraTS 分割**。 |

3. **需要时设置本地 MONAI BraTS 案例。**

   - 在 **1 · 官方访问**中，可以在外部链接确认后打开官方网站。打开网页既不会下载数据，也不代表接受条款。
   - 在 **2 · 选择并校验**中，选择一个包含 T1、T1ce、T2、FLAIR 和 SEG NIfTI 体的本地案例文件夹。校验会在后台开始。修正问题后使用**再次校验**（`Ctrl+R`），或使用**取消校验**安全停止。
   - 在 **3 · 查看校验**中，检查每个输入的状态、形状、间距、方向、SEG 标签计数，以及全部警告和错误。五个输入必须具有兼容几何，SEG 标签也必须符合该任务。
   - 有效的会话内报告即可启用 MONAI 运行，保存清单并非必需。如果要保存，请自行阅读官方条款，选中**已查看官方条款**，再选择**保存清单…**（`Ctrl+S`）。JSON 只含校验和与几何摘要，不含像素、文件名、绝对路径或人员标识。
   - 全体积推理按照 T1ce → T1 → T2 → FLAIR 顺序输入，使用非零体素 z-score 归一化、滑动窗口分块和明确阈值。界面在所选轴向结果切片上显示 WT、TC、ET 概率图和复合掩膜；详情会报告相对 SEG 的 Dice、HD95 和绝对体积误差，以及设备、分块设置、耗时、哈希和领域警告。

4. **运行外部清单。**打开**外部清单**，依次使用编号控件：

   1. 选择 **1. 选择清单…**并打开本地 YAML 清单。执行任何模型代码前，检查会先校验其中声明的身份、任务、运行方式、许可、输入、输出、预处理、能力和本地权重路径。
   2. 阅读**模型资源状态**和兼容性提示。所需权重必须已存在于清单声明的本地路径。如果清单声明了经过审查的 GitHub 来源，**打开官方指南**只会在浏览器中打开该页面；应用不会下载文件。
   3. 选择 **2. 加载模型**。如果清单使用自定义脚本适配器，只有在自行审查适配器后才应同意警告。独立进程可以隔离崩溃与依赖问题，但不是安全沙箱。
   4. 在**影像**中打开并激活目标影像。桌面界面只接受清单声明的一个二维 `image` 输入，并检查输入数量、语义、维度、模态和必需间距。
   5. 如果内存中只有有界普通图像预览，必须明确选中**允许仅预览教学推理**，再核对第二个、与模型相关的确认。结果会被记录为仅预览。
   6. 状态以**就绪：兼容**开头后，选择 **3. 运行推理**。需要停止时选择**停止**以请求取消。
   7. 将**当前模型输入**与每个输出选项卡对照。类型化的掩膜、框和关键点会按源坐标绘制；影像和热图使用声明的或稳定的数值范围；文字和表格类结果显示在摘要中。

### 如何判断成功

- **校验目录与设备**报告 `3/3 个模型已验证`，并显示 CUDA 设备或明确的 CPU 回退。
- 内置模型运行后，操作状态变为**已在本地完成**，**实验**视图和结果详情出现内容。
- 有效 BraTS 案例会报告所有本地输入均通过几何和标签校验；运行按钮会从设置操作变为分割操作。
- 外部清单会依次进入**模型清单已就绪**、**模型已就绪**和兼容的**就绪**状态。推理后，摘要会显示任务、运行方式、时长、输入分辨率和可视化类型。

### 边界与注意事项

- 内置模型是固定教学参考，不是通用临床模型。DIVal 接受经过审查的 LoDoPaB Hann-FBP 领域输入，而不是任意 CT HU 或扫描仪正弦图；DeepInverse 使用从影像派生的模拟数据，而不是扫描仪原始 k 空间；MONAI 在 BraTS 2018 领域训练，尚未验证为通用脑部病变模型或临床系统。
- DeepInverse 合成 MRI 演示目前没有 JSON 导出。DIVal 和已完成的 BraTS 运行可导出不含像素的 JSON 记录。
- 通用外部清单界面无法收集多幅影像、完整三维 / 四维体、提示、k 空间加掩膜、正弦图或其他复合输入。清单即使本身有效，只要不兼容就会保持阻止状态，不会被静默降级。
- 应用绝不会自动获取外部文件、权重、依赖或 URL。缺少本地资源时，**2. 加载模型**保持禁用。
- 外部输出可视化不会修改源影像、创建检查图层，也不提供桌面结果文件导出。

## 评价

### 用途

使用**评估**检查匿名化组是否跨越训练、验证和测试划分；以明确阈值评估一组二分类标签与预测概率；检查校准情况；并导出不含像素的证据记录。

### 前置条件

内置示例概率可以立即评估，数据集清单是可选的。若使用清单，它必须是 1 字节至 4 MiB 的 UTF-8 JSON、YAML 或 YML 文件，使用架构版本 1，明确声明已去标识化，不包含文件系统路径或身份字段，并通过小写 SHA-256 而不是文件名引用数据产物。

最小 JSON 清单示例如下：

```json
{
  "schema": "openmedvisionx-dataset-manifest/v1",
  "schema_version": 1,
  "dataset_id": "demo-dataset",
  "dataset_version": "v1",
  "task": "classification",
  "license_id": "CC-BY-4.0",
  "deidentified": true,
  "label_schema": {},
  "provenance": {},
  "samples": [
    {
      "sample_id": "sample-001",
      "group_id": "group-a",
      "split": "train",
      "artifact_sha256": "aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa",
      "modality": "MR",
      "labels": {"target": 0},
      "site": null,
      "scanner": null,
      "deidentified": true
    }
  ]
}
```

每个 `sample_id` 必须唯一；同一个 `group_id` 只能出现在 `train`、`validation` 或 `test` 中的一个划分。

### 步骤

1. 可选：选择**打开清单…**，打开 `.json`、`.yaml` 或 `.yml`，查看训练、验证和测试的样本数与组数。
2. 在**参考标签**中输入以逗号或换行分隔的 `0` 和 `1`，并确保两个类别都出现。
3. 在**预测概率**中输入相同数量、位于 `[0, 1]` 范围内的有限数值。
4. 设置 0.00–1.00 的**决策阈值**。默认值为 0.50，步长为 0.05；应用不会根据数据自动挑选阈值。
5. 选择**评估**。
6. 查看**指标**：

   - 准确率、敏感度、特异度、PPV、NPV 和 F1 取决于所选阈值。
   - AUROC 和 AUPRC 概括排序表现；AUROC 不会替你选择可部署的工作点。
   - Brier 分数和预期校准误差越低，通常表示概率质量越好。
   - 对工作区支持计算区间的指标，**95% 区间**列会显示结果。

7. 查看**校准**。十个概率区间会显示计数、平均分数和观察到的阳性率；空区间显示 `—`。
8. 选择**导出实验记录…**并保存为 JSON 或 YAML。记录包含输入向量摘要与形状、阈值、指标、可选的数据集清单身份以及手工输入警告，不包含原始标签、原始概率、影像像素或源路径。

### 如何判断成功

- 有效清单会报告**清单校验完成，组泄漏检查已通过。**，并显示数据集 ID、版本和任务。
- 有效评估会报告**评估完成**，并填充十行指标与十行校准结果。
- 只有完成有效评估后，**导出实验记录…**才会启用；导出后状态会报告**实验记录已导出。**

### 边界与注意事项

- 桌面评估表单只支持二分类。分割、检测和配准指标可能出现在各自的专用流程中，但这里没有对应的通用桌面表单。
- 手工输入的数据缺乏可信来源时，需要在其他位置补充记录。导出的摘要可以证明使用了哪组向量，但不能证明标签正确。
- 按组安全划分可以防止一种常见泄漏，却不能消除所有偏差或数据集漂移。
- 指标或置信区间只说明所提供样本中的证据，不是临床主张。

## AI 助手

### 用途

使用 **AI 助手 → 教学对话**向已配置的服务询问概念、参数和学习结果。使用**结构化产物 · API 预览**检查可信宿主集成已经提供的类型化产物契约与本地审阅状态。

### 前置条件

教学对话需要一个服务账户，或者一个有意设置为免认证的本机回环服务。准备准确的模型 ID 和凭据**引用**，不要准备要粘贴到界面的密钥明文。

| 服务 | 默认端点 | 默认凭据引用 |
| --- | --- | --- |
| OpenAI | `https://api.openai.com/v1/responses` | `env:OPENAI_API_KEY` |
| Anthropic | `https://api.anthropic.com/v1/messages` | `env:ANTHROPIC_API_KEY` |
| Moonshot/Kimi | `https://api.moonshot.cn/v1/chat/completions` | `env:MOONSHOT_API_KEY` |
| 智谱 GLM | `https://open.bigmodel.cn/api/paas/v4/chat/completions` | `env:ZHIPU_API_KEY` |
| DeepSeek | `https://api.deepseek.com/chat/completions` | `env:DEEPSEEK_API_KEY` |
| OpenAI 兼容服务 | 将占位内容替换为服务提供的准确端点 | `env:OPENMEDVISIONX_API_KEY` |

有效的凭据引用格式是 `env:NAME`、`keyring:<service>/<username>` 和 `none`。只有 `localhost`、以 `.localhost` 结尾的名称或回环 IP 可以使用 `none`。绝不要把 API 密钥粘贴到**凭据引用**字段。

如需附加影像，请先在**影像**中打开数据、激活所需平面，并检查其中是否存在烧录在像素内的隐私文字。本版本将 DeepSeek 配置为纯文本服务，因此其**视觉输入**和影像附加控件处于禁用状态。

### 步骤

1. 打开**教学对话**，在**服务配置**中：

   - 选择**服务**；
   - 输入该服务提供的准确**模型 ID**；
   - 检查或修改**端点**；
   - 输入不含密钥明文的**凭据引用**；
   - 只有准备发出请求时才选中**启用网络**。

2. 发送纯文本请求时，不要选中**附加当前活动渲染平面**。输入学习问题并选择**发送**，或按 `Ctrl+Enter` / `Cmd+Enter`。只按 `Enter` 会换行。服务配置完成后可以选择**隐藏设置**。

3. 发送带影像的请求时：

   - 先选中**视觉输入**，再选中**附加当前活动渲染平面**；
   - 确认**云端影像传输**从**关闭**变为**开启——本次确切请求需要逐次审阅**；
   - 选择**发送**，检查**审阅将要发送的确切请求**；
   - 将可见 PNG 与目标平面对照，并逐项检查服务、目标主机、端点、模型 ID、任务、总字节数、提示指纹、尺寸、MIME 类型、项目 SHA-256、变换、去标识化操作、像素文字检查和残余风险；
   - 只有所有字段和像素都可接受时，才选择**仅授权本次并发送**。默认选择是**取消**。

4. 请求运行时，选择**取消请求**或按 `Esc` 可请求安全取消。此时仍可继续编辑文字草稿，但服务设置会保持锁定，直到请求结束。

5. 阅读回复。服务文字会以安全 Markdown 渲染，服务名称、模型、时间戳和免责声明单独显示。选择 HTTP 或 HTTPS 链接前，应用会要求确认是否打开浏览器；其他链接协议会被阻止。

6. 需要了解类型化输出契约时，打开**结构化产物 · API 预览**。选择器包含文字解释、类别分数、语义标签、二维掩膜、三维掩膜、重建影像和重建体。除非可信宿主集成同时提供相互匹配的类型化请求与响应，否则面板保持空白。当未经校验的响应与请求匹配时，**确认产物**和**拒绝产物**才会启用。两种操作都只记录本地审阅，不会编辑源数据或创建图层。

### 如何判断成功

- 发送前，可用性提示变为**就绪：请核对服务详情，然后发送此学习请求。**
- 请求期间，标题显示**正在生成新回复…**，取消控件启用。
- 完成后状态显示**回复已就绪 · Markdown 已安全渲染。**，并显示服务 / 模型 / 时间元数据和服务免责声明。
- 发送影像时，状态会报告已审阅的单次请求正在进行；修改任何已绑定字段都必须重新确认。
- 在结构化产物中，匹配但未经校验的响应会报告需要明确审阅；完成后显示**用户已确认**或**已拒绝**。

### 边界与注意事项

- **启用网络**和**附加当前活动渲染平面**是两个独立授权。即使不附加影像，文字提示仍会发送，因此应先删除姓名、标识符和其他隐私内容。
- 附加的 PNG 是经过显示映射后的完整活动二维平面的重新编码；不含视口缩放 / 平移、测量、十字线、掩膜、轮廓或标注。它不是原始 DICOM/NIfTI，也不携带原始元数据，但烧录在像素内的标识仍然存在。
- 授权只对确切的服务、端点、模型、提示、任务和字节生效一次；已经完成的请求无法召回。
- Markdown 只是显示内容，不是可信建议。打开外部链接需要单独决定。
- 普通**教学对话**回复不会转成**结构化产物**。本版本没有桌面产物导入器；结构化审阅面板不会发送内容、不会创建图层，也不能证明产物具备临床有效性。
- 启用影像传输前，请阅读 [AI 服务与云端影像安全](LLM_SECURITY.zh-CN.md)。

## 安全工作核对清单

- 优先使用合成数据、公开数据或所在机构批准的数据。
- 执行 CT、外部模型、ROI 或助手操作前，确认活动视图正确。
- 区分显示映射与解码数值，也要区分像素单位与来自可信来源的物理间距。
- 将平面局部对照与有参考关系的临床图层分开管理。
- 解释输出前，阅读每个模型的输入契约、模型卡、领域警告和设备回退信息。
